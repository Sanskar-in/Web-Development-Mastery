Architecture & Tech Stack

HTML Full Mastery is engineered to be blazing fast, highly portable, and entirely serverless. Unlike modern web applications that rely heavily on heavy build steps (Webpack, Vite) and virtual DOMs (React, Vue), this platform takes a "back to basics" approach with steroids.

🟨

Vanilla Engine

No frameworks. The entire rendering, routing, and search engine is built in pure ES6+ JavaScript, resulting in zero overhead.

🎨

CSS3 Custom Properties

Theming is handled natively by the browser using CSS variables, allowing for instant, jank-free toggling between light, dark, and 25+ premium themes.

💾

Local-First Storage

User progress, flashcard statistics, and custom code editor contents are persisted securely on the client using localStorage.

Modular Setup & CDN Fallbacks

To keep the application highly organized, the codebase is modularized. However, to fulfill the requirement that a user can share a single HTML file that works flawlessly, we implemented an advanced "Self-Healing CDN Fallback".

💡
How it works: If you open the local HTML Mastery.html file without the accompanying CSS/JS files, the browser fires an onerror event. We catch this and instantly fetch the missing assets from the jsDelivr GitHub CDN.
📁 Web Development Mastery
📄 index.html (Main Entry & Self-Healer)
📄 style.css
📄 script.js
📄 developer.html (This documentation)

Here is the implementation of the fallback mechanism found in the <head> of the application:

HTML Fallback Injection
<link rel="stylesheet" href="style.css" onerror="this.onerror=null;this.href='https://cdn.jsdelivr.net/gh/Sanskar-in/Web-Development-Mastery@main/Web%20Development%20Mastery/HTML/style.css';">

URL Routing API (Deep Linking)

HTML Mastery operates as a Single Page Application (SPA). To link to specific tags from external documentation or your own websites, use the URL Hash API.

Standard Hash Linking

Append #tagname-div to the URL to instantly scroll to and highlight a specific tag.

URL Structure
https://yourdomain.com/HTML Mastery.html#canvas-div https://yourdomain.com/HTML Mastery.html#video-div

When the application detects a hash on load, it executes the following DOM scroll routine:

JavaScript Engine (Internal)
window.addEventListener('load', () => { if (window.location.hash) { const target = document.querySelector(window.location.hash); if (target) { target.scrollIntoView({ behavior: 'smooth', block: 'center' }); target.classList.add('highlight-pulse'); } } });

Advanced Theming API

The theme engine is completely detached from the JavaScript logic. It relies purely on redefining CSS root variables based on a data-theme attribute on the <html> tag.

You can programmatically inject custom themes via the browser console or extensions by modifying the CSS variables on the root document:

JavaScript Custom Theme Injection
// Inject a neon cyberpunk theme programmatically const root = document.documentElement; root.style.setProperty('--bg', '#050510'); root.style.setProperty('--bg2', '#0a0a20'); root.style.setProperty('--text', '#00ffcc'); root.style.setProperty('--accent', '#ff00ff'); root.style.setProperty('--border', '#00ffcc'); // Save custom settings localStorage.setItem('htmlMasteryCustomTheme', 'cyberpunk');

State Management Schema

We use standard localStorage to maintain state without requiring a backend database. Below are the primary keys and their expected JSON schemas used by the application.

  • htmlMasteryTheme (String) - Represents the active theme identifier (e.g., "dark", "light", "monokai").
  • htmlMasteredTags (Array) - Tracks user mastery progress.
JSON Schema: htmlMasteredTags
[ "div", "canvas", "audio", "template" ]
⚠️
Data Deletion: Running localStorage.clear() will irrevocably wipe all user progress, saved code editor snippets, and theme configurations.

Performance Optimization Guidelines

The platform maintains a strict 100/100 Lighthouse score. If you are contributing code, you must adhere to the following performance guidelines:

  • No Reflows: Use transform and opacity for all animations. Never animate width, height, or margin.
  • Event Delegation: Do not attach event listeners to individual items in lists (e.g., the sidebar tags). Attach a single listener to the parent container and use e.target to determine the clicked element.
  • Defer Scripts: Any external scripts (like CodeMirror) must be loaded dynamically only when the user clicks the "Try It" button to save initial payload size.

Contribution Guidelines & Workflow

HTML Full Mastery is an open-source initiative. We welcome contributions from the community, whether it's fixing typos, adding new tag explanations, or building new features.

How to Contribute

  1. Fork the Repository: Fork SanskarYadav2012/Web-Development-Mastery on GitHub.
  2. Clone Locally: You do not need to run npm install. Just clone and open HTML Mastery.html in your browser.
  3. Make Changes: Add your changes. Ensure you test in both dark and light modes.
  4. Submit a Pull Request: Open a PR with a detailed description of your changes.

Code of Conduct: We are committed to providing a welcoming and inspiring community for all. Harassment, discrimination, or abusive behavior in PRs or Issues will not be tolerated.
