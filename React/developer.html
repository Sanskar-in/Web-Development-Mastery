Code Philosophy

The principles that guide every line of code in this project.

✨

Clean Code

Every component is written with clarity, consistency, and maintainability in mind. Well-structured code, meaningful naming conventions, proper documentation, and separation of concerns make the codebase a joy to read and contribute to.

🌍

Open Source

This entire platform is open-source and free forever. We believe knowledge should be accessible to everyone regardless of background or financial situation. Community contributions are welcomed and encouraged — every PR makes us better.

Every design decision prioritizes the learning experience. Interactive examples, clear explanations, progressive difficulty, and hands-on practice ensure that concepts don't just get read — they get understood, applied, and remembered.

🤝 Contribute

Contributing Guide

Help make React Mastery better. Here's how to get started contributing.

01

Fork the Repository

Head over to github.com/Sanskar-in/Web-Development-Mastery and click the Fork button in the top-right corner to create your own copy of the repository.

02

Clone Your Fork

Clone your forked repository to your local machine using git clone https://github.com/YOUR-USERNAME/Web-Development-Mastery.git and navigate into the project directory.

03

Create a Feature Branch

Create a new branch for your changes with git checkout -b feature/your-feature-name. Use descriptive branch names that reflect the changes you're making.

04

Make Your Changes

Implement your changes, following the existing code style and conventions. Test your changes locally by opening the HTML files in a browser. Ensure your changes are responsive and work across different themes.

05

Commit & Push

Stage your changes with git add ., commit with a clear message using git commit -m "feat: add your feature description", and push to your fork with git push origin feature/your-feature-name.

06

Open a Pull Request

Navigate to the original repository on GitHub and open a Pull Request from your branch. Provide a clear description of your changes, screenshots if applicable, and reference any related issues. We'll review it promptly!
